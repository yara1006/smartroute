# SmartRoute Design

## 1. 设计定位

SmartRoute 的移动端不是独立 App 首页，而是美团 App 内嵌路线规划插件。视觉上要像美团原生功能：轻量、直接、可执行，以地图、路线卡和时间轴为核心。

设计目标：

- 用户一眼知道“这是美团帮我排好的路线”。
- 用户不需要理解算法，只需要看到下一步去哪、几点到、花多少钱、会不会排队。
- 用户可以用自然语言继续调整路线，而不是回到起点重新输入。

## 2. Web 展示框架

最终 Web Demo 采用三栏结构：

```text
左侧：入口场景选择器
中间：美团手机模拟器
右侧：SmartRoute Agent 解释面板
```

左侧入口场景：

- 搜索页：模拟美团搜索页，用户搜索区域/品类后，结果区出现 SmartRoute 卡片。
- 问小团：模拟小团 AI 通用提问页，用户输入自然需求后，由意图识别命中 SmartRoute，再在回复中嵌入插件卡。
- 收藏夹：模拟收藏夹列表，用户勾选 2-5 个商户后，顶部出现“一键排路线”操作，所选商户作为 pinned stops。
- POI 详情页：模拟商户详情页，出现“从这里出发/加入路线”操作，并把当前商户坐标作为路线锚点。

中间手机模拟器：

- 不展示独立 SmartRoute 首页。
- 先展示美团原生页面上下文，再展示 SmartRoute 插件入口。
- 用户点击入口后进入统一的 SmartRoute 插件路线页。

右侧解释面板：

- 当前入口：搜索页 / 问小团 / 收藏夹 / POI 详情页。
- 输入上下文：自然语言、地图锚点、收藏 POI、搜索结果、当前 POI。
- Agent 链路：路线意图识别 → 解析约束 → 检索 POI → 规划路线 → 约束解释 → 反馈记忆。
- 评分点命中：>=3 POI、餐饮 + 文化/娱乐、多方案、实时调整、<10 秒。

## 3. 移动端使用动线

推荐动线：

1. 美团场景页：搜索页、问小团、收藏夹或 POI 详情页。
2. SmartRoute 入口卡：提示“已发现可串联地点/可从这里出发/可一键排路线”。
3. 插件路线结果：地图 + 底部路线卡 + 多方案切换。
4. 路线详情：时间轴、每站理由、预算/等待/距离解释。
5. 实时调整：底部固定输入框，支持“少走路”“换便宜点”“加晚餐”。
6. 调整反馈：展示调整状态、前后指标变化、站点变化；无法改善时给出放宽建议。
7. 反馈记忆：喜欢/不合适原因沉淀到用户画像。

四类入口的手机端表现：

- 搜索页：搜索结果下方插入 SmartRoute 卡片，例如“已根据深圳大学附近生成 3 小时不绕路路线”。
- 问小团：首屏仍是通用小团输入框，不默认填路线规划 prompt；当用户输入“深圳大学附近下午 3 小时”“外滩下午 3 小时”“今晚想吃饭再逛逛”等路线意图后，小团回复 SmartRoute 插件卡，例如“已生成 2 条路线：低排队 / 文艺体验”。
- 收藏夹：收藏列表顶部插入 SmartRoute 卡片，例如“已选择 3 个地点，按距离、营业时间、排队情况排成路线”。
- POI 详情页：商户信息区增加操作，例如“以这里为起点，继续安排下一站”。

问小团入口的交互原则：

- 输入框 placeholder 保持通用，例如“附近有什么好吃的？”“想去哪玩？”，不写成“帮我规划路线”。
- 默认推荐词不强行占用路线规划，只在“试试这样问”中保留少量自然表达，例如“外滩下午 3 小时怎么玩”。
- 路线意图高置信时直接出现 SmartRoute 插件卡。
- 路线意图中置信时先出现确认卡：“要不要帮你排成一条路线？”操作为“排路线”和“只看推荐”。
- 小团给出单点推荐后，也可以在推荐列表底部提供次级入口：“把这些地点排成路线”。

移动端当前问题：

- 当前手机壳已经改为美团内嵌插件动线，但信息密度仍偏高，答辩时适合展示，真实移动端需要再收敛层级。
- 当前已展示“来自搜索页/问小团/收藏夹/POI详情页”的调起上下文。
- 当前已支持根据入口传入不同地图锚点：深圳大学、外滩、收藏夹已选店铺、POI 详情页当前商户。
- 当前已支持模拟画像/脱敏真实画像切换、脱敏画像 JSON 导入、追问卡、局部调整和调整结果说明，但替换 POI 仍是桌面侧栏，后续应改为移动端底部抽屉。

下一步移动端重构方向：

- 路线结果页采用真实 App 常见结构：上半屏地图，下半屏路线浮层。
- 替换 POI 做成底部抽屉，而不是桌面式侧栏。
- 将画像解释、约束冲突和调整历史收进可展开模块，减少首屏压力。

## 4. 桌面端设计

桌面端从单一 Demo Cockpit 升级为“场景模拟器 + Agent 解释面板”，用于答辩解释系统能力。

桌面端信息结构：

- 左侧：四个入口场景选择器，以及当前场景的用户上下文。
- 中间：美团手机模拟器，展示原生页面、SmartRoute 入口卡和插件路线页。
- 右侧：生成过程、候选 POI、路线解释、约束命中和评分点映射。

桌面端不是最终用户主形态，而是评委和队友理解产品逻辑的展示界面。

## 5. 视觉规范

品牌感：

- 主色：美团黄 `#FFD100`。
- 文字：黑色和深灰，避免大面积彩色渐变。
- 背景：浅灰、白色卡片。
- 地图：使用高德真实地图，marker 保持美团黄 + 黑字。

核心组件：

- Route Card：路线标题、可信度、总时长、人均、等待。
- Map Panel：高德底图、编号 marker、路线 polyline。
- POI Timeline：站点序号、到达/离开时间、POI 名称、评分、人均、等位。
- Constraint Chip：预算、排队、步行、区域、画像。
- Explain Panel：为什么选、为什么没选、冲突如何处理。
- Profile Influence：展示搜索偏好、收藏偏好、浏览标签、预算和排队容忍度如何影响路线。
- Profile Import：在画像区展示“模拟画像 / 脱敏真实画像”tab，支持选择样本和粘贴脱敏 JSON。
- Adjust Input：底部固定自然语言输入。
- Adjustment Result：展示 `applied / partial / not_applied`、等待/人均/移动 delta、站点变化和放宽建议。
- Replace Drawer：同类 POI 替换，展示预算/等待/距离变化。

## 6. 页面状态

必须覆盖：

- 初始状态：展示入口来源和示例需求。
- 规划中：展示“解析约束、检索 POI、组合路线、生成解释”。
- 成功状态：地图、路线卡、时间轴、候选 POI、解释完整展示。
- 空状态：当前约束下无可用路线，提示放宽预算/等待/区域。
- 错误状态：API 报错或地图加载失败时不能白屏。
- 降级状态：高德 Key 缺失或域名限制时，使用本地路线示意图。
- 动态锚点状态：展示当前路线围绕哪个地点生成，避免看起来像固定 Demo。
- 画像切换加载态：展示“画像切换中”，避免用户误以为路线未响应。
- 脱敏画像导入态：展示“导入脱敏画像”，导入完成后自动切换并重新规划。
- 局部调整加载态：展示“局部调整中”，完成后展示调整状态和指标变化。
- 调整失败状态：展示 `not_applied` 和 suggested relaxations，不能只给成功文案。

## 7. 文案原则

- 不说“推荐几个地方”，而说“已排好一条可执行路线”。
- 不说“算法认为”，而说“因为你不想排队，所以避开了高等待 POI”。
- 冲突要透明：预算超了、时间不够、排队风险高时要直接提示。
- 调整要轻量：用户可以说一句话，不需要重新填写表单。

## 8. UI 状态

已完成：

- Web 三栏场景模拟器。
- 搜索页、问小团、收藏夹、POI 详情页四个入口。
- 原生美团页面 → SmartRoute 插件入口 → 路线页。
- 搜索/小团按用户输入地点调起路线，收藏夹可勾选 POI，详情页从当前商户坐标出发。
- 问小团 `open_plugin / ask_confirm / normal_answer` 三态。
- 实时调整输入栏、调整历史、画像切换、冲突解释、规划耗时。
- 画像影响链路、动态追问卡、调整前后指标变化、调整失败补救建议。
- 脱敏画像来源切换、样本选择和 JSON 导入面板。

后续：

- 将替换 POI 改为移动端底部抽屉。
- 将路线详情做成上半屏地图 + 下半屏可拖拽浮层。
- 增加前端自动化截图验收。
